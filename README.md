AQUAMY — Agricultural and Aquatic Muirungi Youth Self-Help Group
Digital Financial Management Platform for Kenyan Chamas
AQUAMY is a high-integrity, full-stack web application designed to digitize the financial and administrative operations of a registered Kenyan Self-Help Group. By encoding the group's official constitution directly into the application logic, the system automates member onboarding, monthly contributions, loan lifecycles, and meeting attendance while ensuring 100% compliance with group by-laws.

🚀 Features
🏦 Financial Automation
M-Pesa Integration: Seamless payment processing via Safaricom Daraja API (STK Push).

Contribution Tracking: Automatic detection of partial payments and calculation of late penalties (KES 100/month).

Share Management: Dynamic share valuation and immutable transaction history.

Loan Lifecycle: Full workflow from application and committee voting (with quorum logic) to disbursement and repayment tracking.

👥 Member & Administrative Governance
Role-Based Access (RBAC): 9 distinct roles including Chairman, Treasurer, Secretary, and Auditor.

Constitutional Enforcement: Hard-coded age limits (18–35), registration fees, and loan guarantor requirements.

Meeting Management: Digital attendance tracking that automatically triggers constitutional fines for lateness or absence.

Invite-Only Onboarding: Secure registration via single-use invite codes generated by the Secretary.

📊 Real-Time Analytics
Member Dashboard: Instant view of Net Standing (Contributions + Shares - Loans).

Admin Dashboard: KPI cards for Group Treasury, Loan-to-Treasury ratios, and monthly compliance trends.

Audit Logs: Immutable records of all sensitive operations (waivers, role changes, approvals).

🛠️ Technical Stack
Layer	Technology
Frontend	Next.js 14 (App Router), Tailwind CSS
Backend	Next.js Server Actions (Type-safe mutations)
Database	PostgreSQL (Supabase)
ORM	Prisma
Auth	NextAuth.js (JWT Strategy)
Payments	Safaricom Daraja API
Infrastructure	Docker, AWS EC2
📂 Database Schema Overview
The system uses a highly relational schema to maintain data integrity across 18 models. Key entities include:

Users: Manages lifecycle status (PENDING → ACTIVE → SUSPENDED).

LoanPolicy: Versioned policies to ensure interest rates remain consistent for the life of a loan.

GroupLedgerEntry: An append-only, immutable ledger for the group treasury.

MemberFinancialSummary: Pre-computed aggregates for high-performance dashboard rendering.

⚙️ Local Development
1. Prerequisites
Node.js 18+

Docker (optional, for local DB)

Supabase Account (or local Postgres)

2. Installation
Bash
# Clone the repository
git clone https://github.com/NIVLEKI/aquamy-portal.git

# Install dependencies
npm install
3. Environment Setup
Create a .env file in the root directory:

Code snippet
DATABASE_URL="your-supabase-url"
DIRECT_URL="your-supabase-direct-url"
NEXTAUTH_SECRET="your-secret"
NEXTAUTH_URL="http://localhost:3000"

# M-Pesa Credentials
MPESA_CONSUMER_KEY="your-key"
MPESA_CONSUMER_SECRET="your-secret"
MPESA_PASSKEY="your-passkey"
# ... other vars
4. Database Migration
Bash
npx prisma migrate dev
5. Run Locally
Bash
npm run dev
🔒 Security Design
Bcrypt.js: Passwords hashed with a cost factor of 12.

Middleware Protection: Centralized route guarding for /dashboard and /admin groups.

Database Transactions: All multi-step financial operations use prisma.$transaction() to prevent data drift.

Policy Snapshots: Loan terms are frozen at the point of approval, protecting against retroactive policy changes.
